#gsecu_v0.1

1. 개요.
  gsecu_v0.1은 가장 기본적인 Spring Security 설정을 하고 이를 통해 가장 간단한 인증을 제공한다.

2. 목표.
  1) 인증의 설명.
    우리가 흔히 말하는 로그인은 특정 서비스의 개인 인증을 뜻한다.
    인증을 한 사용자는 인증과 동시에 사용권한을 부여받는다.
    즉, 인증(로그인)처리란 인증과 권한이 모두 일어나는 것이고 이를 처리하는 것을 뜻한다.
  2) Spring Security를 통한 인증 및 권한 설정.
    우리는 최고관리자, 중간관리자, 실무관리자, 사용자의 4가지 그룹으로 권한을 나누고,
    4가지 그룹별로 각각의 사용자를 두어 인증을 설정할 것이다.
      
3. 주요내용.
  3.1. 설정.
    1) web.xml
      Spring Security는 기본적으로 웹보안을 목표로 하는 것 같다.
      물론, method보안도 지원하고 있는 것 같다만, 주요 목표는 웹보안.
      Spring 웹의 핵심인 Spring MVC의 메인은 DispatcherServlet이다.
      고로, Spring Security는 web.xml에 필터로 등록해서 DispatcherServlet의 필터로 작동하게 된다.

    2) security-context.xml
      Spring Security의 본격적인 설정을 하는 파일이다.
      그렇다면 어떤 설정을 할 것인가?
      로그인은 인증과 권한의 두가지가 함께라고 하였다.
      인증에 대해서는 사용자별 id와 password를 설정할 것이고,
      권한에 대해서는 권한별 접근을 설정할 것이다.
